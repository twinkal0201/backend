AUTHENTICATION AND AUTHORIZATION

1. AUTHENTICATION

Authentication means verifying who the user is.

It answers the question:
"Who are you?"

Example:

* User opens Instagram.
* User enters email and password.
* Server checks whether the credentials are correct.
* If correct, the user is authenticated.

Real-life Example:

* Entering your ATM card and PIN.
* The bank verifies your identity.
* This is Authentication.

---

2. AUTHORIZATION

Authorization means checking what an authenticated user is allowed to do.

It answers the question:
"What are you allowed to do?"

Example:

* After logging into Instagram:

  * You can create posts.
  * You can like posts.
  * You can edit your own profile.
  * You cannot delete another user's account.

Real-life Example:

* After entering a company building:

  * Employee can enter employee areas.
  * Manager can access management rooms.
  * Security decides which rooms each person can access.
* This is Authorization.

---

3. DIFFERENCE

Authentication:

* Verifies identity.
* Happens first.
* Example: Login with email and password.

Authorization:

* Verifies permissions.
* Happens after authentication.
* Example: Check whether user is Admin or Customer.

---

4. WEB APPLICATION EXAMPLE

Step 1: Login
User -> Email + Password -> Server

Server verifies credentials.

If valid:

* Generate JWT Token.
* Send token to browser.

This is Authentication.

---

Step 2: Access Protected Route

User requests:

GET /users

Browser sends JWT Token.

Server checks:

* Is token valid?
* Is user logged in?

If yes:

* Allow access.

This is Authentication.

---

Step 3: Admin Route

User requests:

DELETE /users/5

Server checks:

* Is token valid?
* Does user have ADMIN role?

If role = ADMIN:

* Allow deletion.

If role = USER:

* Return 403 Forbidden.

This is Authorization.

---

NESTJS EXAMPLE

JwtAuthGuard

Purpose:
Checks whether user is logged in.

@Get('profile')
@UseGuards(JwtAuthGuard)
getProfile() {
return "Profile Data";
}

If token is invalid:
401 Unauthorized

This is Authentication.

---

RolesGuard

@Roles('admin')
@UseGuards(JwtAuthGuard, RolesGuard)
@Delete(':id')
deleteUser() {
return "User Deleted";
}

Checks:

1. User logged in?
2. User role is admin?

If role is not admin:
403 Forbidden

This is Authorization.

---

SUMMARY

Authentication:
"Who are you?"
Example: Login with email and password.

Authorization:
"What can you do?"
Example: Only admin can delete users.

Authentication comes FIRST.
Authorization comes SECOND.
